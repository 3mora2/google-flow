Access instructions

The child node homepage only retains the running status display, which is used by the deployer to confirm whether the current instance role, browser capacity, and health status are normal.

Node responsibilities

  • Perform local Playwright + Chromium coding.
  • Register with the master node and continuously report heartbeats.
  • Receive routed session finish / error callback.

Troubleshooting entrance

  • /api/v1/health Check the instance survival status.
  • /admin View configuration, logs, and node execution information.
  • /api/portal/summary View public run summary.

If the role shown here is not subnode,Check, please FCS_CLUSTER_ROLE or data/setting.toml。
